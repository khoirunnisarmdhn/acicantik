4. Fungsi Controller, Model, dan Resource (Views)

Controller

"Otak" aplikasi. Menerima request dari user, memproses data menggunakan Model, lalu mengembalikan tampilan (View) ke browser. Semua logika bisnis ada di sini.

Model

Representasi tabel di database. Digunakan untuk query data. Contoh: Kas::all() artinya ambil semua data dari tabel kas. Di aplikasi ini sebagian besar menggunakan DB::table() langsung.

Resource / Views

Folder resources/views/ berisi semua file tampilan .blade.php. Disebut "resource" karena ini sumber daya tampilan aplikasi. Setiap halaman web punya file tampilan tersendiri (index = daftar, create = form tambah, edit = form ubah, show = detail).

6. Kenapa Dashboard Tidak Ada di Database?

Dashboard tidak punya data sendiri — dia hanya membaca dan merangkum data dari tabel-tabel lain (kas, proyek, dll) lalu menampilkannya sebagai ringkasan. Tidak perlu tabel khusus karena tidak ada data baru yang disimpan.

Sedangkan tipe_termin, metode_bayar, rincian ada di database karena mereka menyimpan data yang dibutuhkan oleh transaksi dan perlu di-query saat form diisi.

8. Pertanyaan Umum Lain yang Sering Ditanya

PertanyaanJawaban Singkat
Apa perbedaan sistem dan aplikasi?Sistem = keseluruhan proses bisnis. Aplikasi = alat digital untuk menjalankan sistem tersebut.
Mengapa pakai Laravel?Laravel menyediakan struktur MVC yang rapi, ORM Eloquent untuk database, sistem routing, dan keamanan bawaan sehingga mempercepat pengembangan.
Apa itu Blade template?Template engine milik Laravel untuk membuat tampilan HTML dengan sintaks PHP yang lebih sederhana. File berekstensi .blade.php.
Apa itu migration?Script PHP untuk membuat/mengubah struktur tabel database secara terprogram, sehingga perubahan database bisa di-track dan diulang.
Apa itu route?Aturan yang menghubungkan URL dengan Controller. Contoh: URL /kas-masuk diarahkan ke KasMasukController@index.
Kenapa ada session di Laravel?Untuk menyimpan data sementara di server (seperti status login, pesan notifikasi) yang hilang setelah browser ditutup.
Apa fungsi .env?File konfigurasi environment yang menyimpan data sensitif seperti nama database, password, API key. Tidak di-upload ke GitHub.

10. Hal Penting Lain Sebelum Sidang

Keamanan Aplikasi

  • Autentikasi login menggunakan Laravel Breeze
  • Semua halaman dilindungi middleware auth — hanya bisa diakses setelah login
  • Password disimpan dalam bentuk hash bcrypt — tidak bisa dibaca langsung di database
  • Form dilindungi CSRF token — mencegah serangan dari luar

Alur Teknis Saat Klik "Simpan"

  • User klik Simpan → data dikirim via HTTP POST
  • Masuk ke method store() di Controller
  • Data divalidasi terlebih dahulu
  • Jika valid → disimpan ke database
  • Jurnal otomatis dibuat di dalam method store() yang sama
  • Redirect ke halaman index dengan notifikasi SweetAlert2
